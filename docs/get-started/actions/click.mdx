# click

The `click` action allows you to click or tap an element on the page. You can specify which mouse button to use and target elements using text, selectors, ID, classes, attributes, or ARIA name.

The `click` action works in several ways:

- **String Shorthand:** Provide the display text, CSS/XPath selector, ID, `data-testid` value, or ARIA name directly as the value for the `click` key. Doc Detective searches across all these types in parallel and uses the first match found with this precedence: elementText → elementAria → elementId → elementTestId → selector.
- **Object Format:** Use an object with detailed properties for more control over the click action.

If you need to find an element before clicking it, consider using the [`find`](/docs/get-started/actions/find) action, which lets you locate elements using additional criteria and optionally click them in a single step.

**Note:** If you use the `click` action without specifying a target (using element text or selector), it clicks the active element or the element at the current cursor position.

## Properties

When using the object format, you can specify:

- `button`: (Optional) Kind of click to perform. Can be `"left"`, `"right"`, or `"middle"`. Default is `"left"`.
- `selector`: (Optional) [CSS selector](/docs/get-started/selectors/css) or [XPath selector](/docs/get-started/selectors/xpath) of the element to click.
- `elementText`: (Optional) Display text of the element to click.
- `elementId`: (Optional) The ID attribute of the element to click. Supports regex patterns using `/pattern/` syntax.
- `elementTestId`: (Optional) The `data-testid` attribute of the element to click. Supports regex patterns using `/pattern/` syntax.
- `elementClass`: (Optional) Class name or array of class names that the element must have. Supports regex patterns.
- `elementAttribute`: (Optional) Object of attribute key-value pairs that the element must have.
- `elementAria`: (Optional) The computed accessible name of the element. Supports regex patterns using `/pattern/` syntax.

If you specify multiple element finding properties, the element must match all of them. For complete details on element finding, see the [`find`](/docs/get-started/actions/find) action.

> For comprehensive options, see the full [`click`](/docs/references/schemas/click) reference.

## Examples

Here are a few ways you might use the `click` action:

### Click by element text (string shorthand)

```json
{
  "tests": [
    {
      "steps": [
        {
          "description": "Click on an element with text 'Submit'",
          "click": "Submit"
        }
      ]
    }
  ]
}
```

### Click by selector (string shorthand)

```json
{
  "tests": [
    {
      "steps": [
        {
          "description": "Click on an element matching the CSS selector",
          "click": "#submit-button"
        }
      ]
    }
  ]
}
```

### Click an element by text (object format)

```json
{
  "tests": [
    {
      "steps": [
        {
          "description": "Left-click on a button with specific text",
          "click": {
            "elementText": "Submit",
            "button": "left"
          }
        }
      ]
    }
  ]
}
```

### Click an element by selector (object format)

```json
{
  "tests": [
    {
      "steps": [
        {
          "description": "Middle-click on an element with a specific selector",
          "click": {
            "selector": "#open-in-new-tab",
            "button": "middle"
          }
        }
      ]
    }
  ]
}
```

### Click an element by both selector and text

```json
{
  "tests": [
    {
      "steps": [
        {
          "description": "Click on a specific element matching both selector and text",
          "click": {
            "selector": ".btn",
            "elementText": "Download",
            "button": "left"
          }
        }
      ]
    }
  ]
}
```
